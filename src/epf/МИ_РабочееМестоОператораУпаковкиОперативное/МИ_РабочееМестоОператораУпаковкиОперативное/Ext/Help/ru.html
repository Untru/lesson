Рабочее место оператора упаковки

Процедура упаковки может выполняться на стационарном рабочем месте с помощью подключенного сканера считывания штрихкодов.

Для обработки данной операции в зоне контроля и упаковки устанавливаются стационарные рабочие места с подключенными сканерами считывания штрихкодов. Товар свозится в зону контроля, где подлежит сканированию и проверке на полноту потребности заказа и распределению по контейнерам. Процесс упаковки производится по одному Заказу на отгрузку.

Настройка Рабочего стола оператора упаковки осуществляется в Персональных настройках раздела Администрирование.

На закладке Рабочее место оператора упаковки можно указать:

■      количество одновременно обрабатываемых контейнеров;

■      возможность выбора заказа на отгрузку, на основании которого производится упаковка, из списка. При установке флага Выбирать заказ на отгрузку из списка кроме выбора документа сканированием штрихкода возможен выбор по команде из списка документов;

■      возможность выбора контейнера упаковки из списка. При установке флага Выбирать контейнер из списка кроме сканирования штрихкода возможен выбор контейнера из списка контейнеров по команде;

■      статус контейнера упаковки по умолчанию – статус контейнера, формируемых по умолчанию в результате обработки операции упаковки.

Рабочее место оператора упаковки доступно по ссылке панели навигации раздела Отгрузка.

В верхнем окне содержится информация о контейнерах, отобранных по Заказу на отгрузку, с которым ведется работа. В нижнем окне - список конечных контейнеров-получателей товара. Под пиктограммой контейнеров отображается полоса прогресса упаковки контейнера. Аналогичная полоса отображается и для текущего Заказа на отгрузку.

На панели слева содержаться команды для обработки задачи по упаковке. Подсказка текущего действия отображается в нижнем левом углу Рабочего стола.

Для начала работы по упаковке в Рабочем столе оператора упаковки необходимо отсканировать штрихкод Заказа на отгрузку или воспользоваться командой Выберите заказ на отгрузку.

По факту выбора заказа необходимо отсканировать шрихкод контейнера отбора, из которого будет взят товар для контроля и упаковки.

Важно!

Упаковка может выполняться в двух режимах: последовательный контроль и упаковка каждого контейнера, или параллельный контроль нескольких контейнеров. В первом случае сканируется штрихкод первого контейнера с отобранным товаром. Во втором – сканируются все контейнеры отбора по заказу. Количество одновременно обрабатываемых контейнеров задается в Персональных настройках (раздел Администрирование).

В случае невозможности сканирования выбор начального контейнера может быть произведен из списка по кнопке Начальные контейнеры.

Активный контейнер выделяется цветом. В верхней табличной части отображается список товаров, входящих в состав начального контейнера.

По факту выбора начального контейнера необходимо отсканировать конечный контейнер-получатель или выбрать контейнер из списка. Если в процессе упаковки перекладывание товара на другой контейнер не происходит, требуется повторно отсканировать начальный контейнер.

После выбора контейнеров, с которыми необходимо вести работу, сканируются штрихкоды номенклатуры, подлежащей упаковке. По факту сканирования в нижнюю табличную часть переносятся строки с отсканированной номенклатурой.

Для завершения начального контейнера предусмотрена команда Завершить начальный контейнер. Для завершения конечного контейнера предусмотрена команда Завершить конечный контейнер.

В случае необходимости размещения конечных контейнеров на мастер контейнер, необходимо воспользоваться командой Положить в мастер контейнер. Мастер-контейнер может быть выбран сканированием штрихкода или выбором из списка контейнеров.

По факту выполнения упаковки товара с помощью Рабочего места оператора упаковки в системе создаются задачи на перемещение контейнеров/товара с типом Упаковка.

Для просмотра информации по стадии упаковки заказа предусмотрена команда Расшифровка заказа.
